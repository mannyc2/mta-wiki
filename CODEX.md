# Codex Notes

This repository also has detailed operational guidance in `AGENTS.md`. Treat that file as the primary agent instruction surface.

## Keep The Changelog Updated

Before starting substantial work, skim `LOG.md` for recent release milestones, public data-contract
decisions, and active caveats that may affect the task.

Update `LOG.md` whenever work produces any of the following:

- A project decision or reversal.
- A release milestone, publication event, or public data-contract change.
- A new active caveat or known issue that affects public users or future operators.
- A change in recommended commands, workflow, prototype source, or model/provider practice.

Do not use `LOG.md` as a transcript dump, plan archive, or run diary. Keep it changelog-sized,
dated, and durable. Link or name generated artifacts instead of copying large outputs.

When code changes are purely mechanical and do not change project direction or outcomes, no log entry is required.
